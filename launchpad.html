Compiling CPN from Ciphscoin repo are the safest way to access to Ciphscoin.  
These are guide for Ubuntu/Linux based machine.

# Installing dependencies

sudo apt-get update

sudo apt-get install build-essential pkg-config libc6-dev m4 g++-multilib autoconf libtool ncurses-dev unzip git python zlib1g-dev wget bsdmainutils automake libboost-all-dev libssl-dev libprotobuf-dev protobuf-compiler libgtest-dev libqt4-dev libqrencode-dev libdb++-dev ntp ntpdate vim software-properties-common curl libcurl4-gnutls-dev cmake clang

# Install nanomsg

cd ~

git clone https://github.com/nanomsg/nanomsg

cd nanomsg

cmake . -DNN_TESTS=OFF -DNN_ENABLE_DOC=OFF

make -j2

sudo make install

sudo ldconfig

# Compiling CPN Daemon

cd ~

git clone https://github.com/ciphscoin/cpn

cd cpn

git checkout dev

./zcutil/fetch-params.sh

./zcutil/build.sh -j2

# Launching CPN

cd ~

cd cpn/src


Optionally, use Screen to attach CPN terminal, execute:

screen -S cpn

Deploy CPN with exact parameter below, execute:

./komodod -ac_name=CIPHS -ac_supply=6000000 -ac_reward=0,9998452,77000000,129680190,210000000,150000000,100000000 -ac_eras=7 -ac_end=7200,50400,2556000,12780000,12780000,17892000,0 -ac_decay=100000000,100000000,100000000,100000000,100000000,100000000,100000000 -ac_halving=1 -ac_adaptivepow=1 -ac_blocktime=12 -ac_cbmaturity=1000 -ac_sapling=1 -ac_snapshot=7200 -ac_cc=313 -ac_ccenable=225,226,227,228,231,232,234,235,236,237,238,239,240,241,242,243 -addnode=152.89.247.97 -addnode=212.114.52.2 &


If no error seeing, mean the launch has been success.
CTRL A + CTRL D to detach CPN Screen
[screen -r cpn to reattach] 
Now launch Ciphscoin mining using cli:
J represent CPU Core. If 4 then set 4.

./komodo-cli -ac_name=CIPHS setgenerate true J


Retrieve wallet info, execute:

./komodo-cli -ac_name=CIPHS getwalletinfo


Retrieve blockchain info, execute:

./komodo-cli -ac_name=CIPHS getinfo


Retrieve current circulating supply, execute:
Change {HighestBlock} with current block from getinfo

./komodo-cli -ac_name=CIPHS getblocksupply {HighestBlock}


To retrieve all command list, execute:

./komodo-cli -ac_name=CIPHS help

In the command below, you may instruct the script to use only a set number of processors. For example, -j8 instructs the script to use 8 processors, -j4 instructs the script to use 4 processors, etc.

This script can take some time. Once completed, you will find komodod.exe & komodo-cli.exe files inside the src directory.Transfer these 2 executable files to your Windows computer and place the files in a new folder on the Desktop called kmd or any other location you prefer (remember the location and use that). For this guide we are using kmd directory on Desktop.Open a command prompt for the following steps.

notepad "%HOMEPATH%\AppData\Roaming\ciphscoin\komodo.conf"

server=1
txindex=1

addnode=178.63.69.164
addnode=88.198.65.74
addnode=5.9.122.241
addnode=144.76.94.38

After pasting, save and exit Notepad.

#Requirements

OSX (version > 10.11)
Minimum 4GB of free RAM (8GB+ recommended)

#Ensure Command Line Tools are Installed.

Issue the following command in a terminal:

xcode-select --install

#Ensure brew is Installed

We use the software brew to install dependencies. If you have the latest version of brew installed already, you may skip this step.


/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

#Use brew to Install Dependencies

Execute each command separately:

brew update
brew upgrade
brew tap discoteq/discoteq; brew install flock
brew install autoconf autogen automake
brew install gcc@8
brew install binutils
brew install protobuf
brew install coreutils
brew install wget

#Clone the Ciphscoin repository

git clone https://github.com/ciphscoin/cpn

#Fetch the Zcash Parameters:

cd cpn
./zcutil/fetch-params.sh

#Compile Ciphscoin

git checkout dev
./zcutil/build-mac.sh -j8

#Create Configuration File

Create the configuration file in the following directory:

~/Library/Application\ Support/ciphscoin

If the directory does not yet exist, create the directory.

mkdir ~/Library/Application\ Support/ciphscoin

Create the configuration file by entering the following commands in the terminal. Execute each line separately.

echo "rpcuser=komodouser" >> ~/Library/Application\ Support/ciphscoin/komodo.conf
echo "rpcpassword=`head -c 32 /dev/urandom | base64`" >> ~/Library/Application\ Support/
ciphscoin/komodo.conf
echo "txindex=1" >> ~/Library/Application\ Support/
ciphscoin/komodo.conf
echo "bind=127.0.0.1" >> ~/Library/Application\ Support/
ciphscoin/komodo.conf
echo "rpcbind=127.0.0.1" >> ~/Library/Application\ Support/ciphscoin/komodo.conf
echo "addnode=5.9.102.210" >> ~/Library/Application\ Support/ciphscoin/komodo.conf
echo "addnode=78.47.196.146" >> ~/Library/Application\ Support/ciphscoin/komodo.conf
echo "addnode=178.63.69.164" >> ~/Library/Application\ Support/ciphscoin/komodo.conf
echo "addnode=88.198.65.74" >> ~/Library/Application\ Support/ciphscoin/komodo.conf
echo "addnode=5.9.122.241" >> ~/Library/Application\ Support/ciphscoin/komodo.conf
echo "addnode=144.76.94.38" >> ~/Library/Application\ Support/ciphscoin/komodo.conf

#Run Komodo

Once all processes are complete, run the cpn daemon.

cd ~/cpn/src

./ciphs &

#Track the Syncing Progress 
#Using the tail Command

tail -f ~/Library/Application\ Support/ciphscoin/debug.log

#Using komodo-cli and getinfo

cd ~/ciphs/src

./komodo-cli getinfo

When the returned properties of blocks and longestchain are equal to each other, the daemon is finished syncing with the network.

#Backup Your Wallet

~/Library/Application\Support/ciphscoin/wallet.datOne

method to backup this file is to archive a copy of the file.


#Copy the file

cp av~/Library/Application\Support/ciphscoin/wallet.dat ~/wallet.dat

# Rename file

mv~/wallet.dat~/2019-05-17-wallet_backup.dat

# To make archive

tar -czvf ~/2019-05-17-wallet_backup.dat.tgz ~/2019-05-17-wallet_backup.dat
